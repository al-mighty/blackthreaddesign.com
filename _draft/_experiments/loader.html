About

There are many 3d object loaders on the internet, so why create another one? Well, this one uses the loaders directly from the three.js github repo - either plugin type loaders from the examples directory (FBX, DAE, OBJ, MTL and GLTF loaders), or directly from the three.js core (JSON file loaders - bufferGeometryLoader, ObjectLoader, JSONLoader). Current loaders are taken from three.js version .

Plugins from the examples directory have been minimally adapted for use as ES6 modules, but are otherwise unchanged.

This means that if you want to quickly test models for use in a three.js project, you can load them here. Any errors encountered during the load will be displayed above this section.

Controls

Left mouse / touch: Rotate camera

Middle mouse / Pinch: zoom

Right mouse / Two finger touch: Pan camera

Currently supported formats

Three.js native JSON format: Complete scene, or a file containing and geometry or buffer geometry. Files containing only geometries will be rendered in wireframe.

FBX: Binary higher than version 6.1 and ASCII higher than version 7.0

GLTF:.gltf or .glb file, only embedded textures supported for now.

OBJ: Using THREE.OBJLoader2. MTL material file not supported yet.

DAE: Using THREE.ColladaLoader2, so skinned meshes are not yet supported. Textures not yet supported.

Supported texture formats

JPG, PNG, GIF, BMP, DDS

Archive support

Models can be loaded in ZIP format, where the ZIP archive contains a single scene and associated textures. The directory structure of the zip does not matter.

Limitations of archive support:

  • Currently only supports FBX
  • Only one model / scene per zip
  • All textures must have a unique name, duplicate textures will be ignored.
